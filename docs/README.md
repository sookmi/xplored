# XploreD Docs

This folder keeps the small set of internal references that still matter to the current product setup.

## Current Docs

- `ARCHITECTURE.md`: runtime structure, data model, reusable UI layout, and required environment variables
- `BRAND.md`: positioning, audience, information architecture, and curation rules
- `../DEPLOYMENT.md`: Vercel and Supabase deployment checklist

## Documentation Rules

- Keep docs aligned with the live Supabase and Vercel setup
- Prefer updating these files instead of creating one-off planning notes in the project root
- Remove outdated process docs once they stop reflecting the active product workflow
